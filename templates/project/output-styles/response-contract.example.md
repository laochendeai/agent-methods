# Response Contract

- Prefer short, direct answers.
- Use concise technical language.
- Keep outcome and next action visible.
- Avoid adding long background unless the user explicitly asks for it.

# Notes

- Put stable output constraints here, not one-off task instructions.
- If this style is plugin-specific, prefer a namespaced file/name.
- Only use `force-for-plugin` when the plugin truly depends on a fixed response contract.
